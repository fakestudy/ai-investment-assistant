## 已掌握内容

- 工具调用参数需要合并完整后再发送，`tool_call` 应先于对应的 `tool_result`。
- 内部 RPC 与外部 REST API 优化目标不同。
- Protobuf 可以作为多语言唯一契约来源。
- 流式失败应以 `error` 结束，不能再发送成功终态 `done`。
- 当前 `Nginx → Python Agent` 是临时开发链路；最终链路为 `FE → Go BFF → Python AI Application Service`。
- Python 服务的目标边界是完整 AI Application Service，拥有会话、消息、流式恢复、checkpoint、长期记忆和 Agent 执行；Go 拥有登录、鉴权、用户与通用业务。
- 数据所有权应按业务闭环划分，而不是按语言或“所有持久化都归后端”划分。
